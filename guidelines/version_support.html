Expect these to be the most commonly used Python versions. Note that some libraries like NumPy drop support for older versions of Python earlier than their end of life (EOL) as outlined in NEP 29.

Realize that users can still install the older version from PyPI via pip as the package manager (pip). When pip is used, it downloads and installs the most recent version of the library that supports your version of Python. You can enforce a minimum- required Python version within setup.py with:

from setuptools import setup

[...]

setup(name="my_package_name",
      python_requires='>3.6',
[...]
)

This helps pip to know which versions of your library support which versions of Python. You can also impose an upper limit if youâre sure you donât support certain versions of Python. For example, if you only support Python 3.6 through 3.9: python_requires='>=3.6, <3.10'.
